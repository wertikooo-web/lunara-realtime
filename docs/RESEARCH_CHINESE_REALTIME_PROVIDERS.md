PRELIMINARY HYPOTHESES: CHINESE REALTIME PROVIDERS
PATH: docs/RESEARCH_CHINESE_REALTIME_PROVIDERS.md
STATUS: RESEARCH BACKLOG (NOT AN IMPLEMENTATION DECISION)
Этот документ фиксирует первичные рабочие гипотезы по китайским альтернативным провайдерам для контура lunara-realtime. До прохождения практических тестов (Practical Account Gates) данные параметры запрещено использовать как основу для архитектурных решений.

1. CONFIRMED (Подтвержденные факты)
Существование продуктов: Мультимодальные омни-модели семейства Qwen (включая ветку Omni Flash/Plus) существуют, используют архитектуру Thinker-Talker и аппаратно способны вести потоковый реалтайм-диалог (аудио-в-аудио).

Существование альтернатив: У Volcengine (ByteDance) есть стриминговые голосовые решения на базе архитектуры Doubao.

Языковой лимит голосового вывода: Технические отчёты Qwen3-Omni и Qwen3.5-Omni заявляют генерацию речи на 10 языках. Конкретный список языков и его соответствие облачному API требуют отдельной проверки.

2. UNCONFIRMED (Неподтвержденные гипотезы, требующие проверки)
Поддержка румынского языка на выходе: Не подтверждено, входит ли румынский в число поддерживаемых языков speech output. До получения официального списка или живого теста статус: UNKNOWN.

Протокол и эндпоинты: Наличие OpenAI-совместимого WebSocket протокола и точный адрес эндпоинта (предположительно wss://dashscope.aliyuncs.com/...) для реалтайм-омни моделей не подтверждены открытой документацией.

Инфраструктура и задержка: Наличие развернутых омни-реалтайм моделей в европейских дата-центрах Alibaba Cloud (Model Studio) и гипотетическая задержка в 60–90 мс из Молдовы — не проверены.

Ценообразование: Реальная стоимость удержания 1 часа непрерывной WebSocket-сессии (включая аудио-токены ввода/вывода) остается неизвестной.

Политика данных: Конкретные правила хранения, удаления (Data Retention) и использования аудиопотоков для дообучения у обеих платформ не верифицированы.

3. REQUIRES ACCOUNT TEST (План практического гейта)
Для верификации Qwen как потенциального Провайдера №2, сразу после отладки Gemini Live должен быть проведен изолированный 60–90 минутный практический тест:

Регистрация: Создать Alibaba Cloud International account из Молдовы. Проверить, пропускает ли система международные карты и требует ли жесткой верификации (китайский паспорт/ID).

Консоль Model Studio: Найти раздел Real-time Multimodal или Omni-Realtime. Проверить, доступны ли модели для международного аккаунта.

Спецификации: Скопировать из консоли точные Model ID и WS Endpoint. Проверить официальную страницу прайсинга для этих ID.

Языковой тест (Русский): Запустить 5-минутную сессию. Проверить качество распознавания и живость голоса.

Языковой тест (Румынский): Запустить тест на румынском языке. Проверить, способна ли модель генерировать ответ голосом (speech-to-speech) или сессия падает в ошибку / переключается на текстовый чат.

Метрики: Снять реальный latency передачи чанков из Railway до эндпоинта и зафиксировать фактическое списание денег с баланса за 10 минут тестов.

Управление потоком: Проверить работу функции прерывания (interrupt).

Стратегический вердикт
Если румынский speech output отсутствует или неприемлем по качеству, Qwen исключается из кандидатов на основной универсальный provider для румынского сценария Lunara. Возможность применения для русского или других рынков (в качестве дешёвого/fallback провайдера, reasoning-компонента или B2B-решений) оценивается отдельно.

Фокус ветки lunara-realtime целиком переносится на Gemini Live как на единственный приоритет для сборки первого работающего прототипа.
